# Marketplacify — How to Publish a Plugin to Both Marketplaces

Step-by-step checklist for adding a new plugin to the `blairanderson/skills` (Claude Code) marketplace **and** the `blairanderson/skills-openai` (Codex) marketplace.

## Two Repos, Two Formats

| | Claude Code | Codex (OpenAI) |
|---|---|---|
| Repo | `~/dev/skills` | `~/dev/skills-openai` |
| Marketplace manifest | `.claude-plugin/marketplace.json` | `.agents/plugins/marketplace.json` |
| Plugin structure | `plugins/<name>/.claude-plugin/plugin.json` + nested `skills/<name>/SKILL.md` | flat `plugins/<name>/SKILL.md` |
| Extra frontmatter | `version`, `argument-hint` allowed | **not allowed** — stripped on mirror |
| Validator | `bin/validate_plugin` | `scripts/quick_validate.py` |

---

## Required Directory Structure

### Claude (`~/dev/skills`)

```
plugins/<plugin-name>/
├── .claude-plugin/
│   └── plugin.json          # Plugin metadata (name, description, version)
└── skills/
    └── <plugin-name>/
        ├── SKILL.md          # The skill definition (frontmatter + instructions)
        └── (optional files)  # references/, scripts/, etc.
```

### Codex (`~/dev/skills-openai`) — generated by mirror script

```
plugins/<plugin-name>/
└── SKILL.md                 # Frontmatter stripped of version/argument-hint
```

---

## Step 1: Create `plugin.json` (Claude only)

Create `.claude-plugin/plugin.json` inside your plugin directory:

```json
{
  "name": "<plugin-name>",
  "description": "One-line description of what the plugin does",
  "version": "1.0.0"
}
```

- **name**: Must match the directory name (e.g., `plugins/tasks/` → `"name": "tasks"`)
- **description**: Same or similar to the SKILL.md description
- **version**: Semver, start at `1.0.0`

## Step 2: Write `SKILL.md`

Place it at `plugins/<plugin-name>/skills/<plugin-name>/SKILL.md`.

Required frontmatter:

```yaml
---
name: <plugin-name>
description: "When to activate this skill — be specific about trigger conditions"
allowed-tools: Bash, Read, Write, Edit, Glob
---
```

Optional Claude-only frontmatter fields (will be stripped when mirroring to Codex):
- `version`: Skill-level version
- `argument-hint`: Example invocations shown to users (e.g., `'list, show 7T, create 4R "Task Name"'`)

The body contains the full instructions Claude/Codex follows when the skill activates.

## Step 3: Add Entry to Claude `marketplace.json`

Edit `.claude-plugin/marketplace.json` and add an object to the `plugins` array:

```json
{
  "name": "<plugin-name>",
  "source": "./plugins/<plugin-name>",
  "description": "Human-readable description for marketplace browsing",
  "author": {
    "name": "Your Name"
  },
  "category": "<category>",
  "tags": ["tag1", "tag2", "tag3"]
}
```

## Step 4: Add Entry to Codex `marketplace.json`

Edit `~/dev/skills-openai/.agents/plugins/marketplace.json` and add:

```json
{
  "name": "<plugin-name>",
  "source": {
    "source": "local",
    "path": "./plugins/<plugin-name>"
  },
  "policy": {
    "installation": "AVAILABLE",
    "authentication": "ON_INSTALL"
  },
  "category": "<Planning|Development|Productivity>"
}
```

## Step 5: Update `README.md` (Claude repo)

Add the new plugin to the "Available Skills" table in `README.md`:

```markdown
| `<plugin-name>` | Category | Description |
```

## Step 6: Mirror to Codex Repo

Run the mirror script to copy and transform the SKILL.md to `skills-openai`:

```bash
.claude/skills/marketplacify/bin/mirror_to_codex <plugin-name>
```

This strips `version` and `argument-hint` from frontmatter (Codex rejects them), then writes
the plugin to `~/dev/skills-openai/plugins/<plugin-name>/SKILL.md`.

## Step 7: Validate Both Platforms

### Claude validator
```bash
.claude/skills/marketplacify/bin/validate_plugin <plugin-name>
# or
.claude/skills/marketplacify/bin/validate_plugin --all
```

### Codex validator (run from skills-openai)
```bash
cd ~/dev/skills-openai
python3 scripts/quick_validate.py plugins/<plugin-name>
# or
python3 scripts/quick_validate.py --all
```

Fix any `[FAIL]` lines before proceeding.

## Step 8: Bump the Marketplace Version (Claude)

Run `/bump` from the Claude repo to increment the version and push to master.

## Step 9: Commit Both Repos

```bash
# Claude repo
cd ~/dev/skills && git add -A && git commit -m "feat: add <plugin-name> plugin"

# Codex repo
cd ~/dev/skills-openai && git add -A && git commit -m "feat: mirror <plugin-name> plugin"
```

---

## Checklist (copy-paste for new plugins)

```
- [ ] plugins/<name>/.claude-plugin/plugin.json — name, description, version
- [ ] plugins/<name>/skills/<name>/SKILL.md — valid frontmatter + body
- [ ] .claude-plugin/marketplace.json — entry added with source, author, category, tags
- [ ] skills-openai/.agents/plugins/marketplace.json — entry added with source object + policy
- [ ] README.md "Available Skills" table updated
- [ ] bin/mirror_to_codex <name> — ran successfully
- [ ] bin/validate_plugin <name> — all PASS (Claude)
- [ ] python3 scripts/quick_validate.py plugins/<name> — PASS (Codex)
- [ ] /bump — marketplace version incremented
- [ ] Both repos committed and pushed
```
